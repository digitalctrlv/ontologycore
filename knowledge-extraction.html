5.2 Knowledge Graph

To represent the ontology visually and make its structure more intuitive, we visualized the OWL model exported from Protégé as RDF triples and explored several graph representations. Because the entire ontology contains a large amount of information, a single static diagram is often too dense to read. We therefore produced multiple visualizations using different tools and perspectives to highlight specific parts of the model.

Figure 1 : OntoGraph view focused on the core entity Aesthetic. The diagram shows classes directly connected to Aesthetic and related subclasses (Community, Context, Emotion, AestheticElement, etc.), giving a quick overview of the main building blocks that form an aesthetic.

OntoGraph gives a useful overview, but it cannot display the names of object properties directly on each link. To show concrete property names and make relationships explicit, we exported OntoGraph to a directed-graph (.dot) format and rendered it with another tool. The result shows labeled arrows that indicate which object properties connect the main classes to Aesthetic.

Figure 2 : The same core classes rendered with object-property labels on the edges. This view clarifies, for example, how Context and ConceptualIdea shape an aesthetic, and how AestheticElement and LifeStyle concretely embody it.

Next, we focused on the creative-process perspective (as described in Section 4.4). We modelled aesthetic diffusion as a three-stage creative process — Divergence, Convergence, and Metaconvergence — and used graph links (e.g., resultsIn) to show how these stages produce or stabilize an aesthetic. The cycle representation also suggests how one aesthetic can give rise to or influence another.

Figure 3 : Visualization of the creative-process cycle. Nodes represent process classes and edges show relations (e.g., resultsIn) that link stages to aesthetic outcomes and to other aesthetics.

We then loaded our dataset of instantiated aesthetics and attempted instance-level visualizations. The example below shows elements related to Dark Academia (outside the creative-process subgraph). The purple diamonds are instances and the ochre dots indicate their class/cluster type. Although this instance graph is somewhat dense, it helps reveal which aesthetic elements, emotions, and behaviours concretely characterize Dark Academia.

Figure 4 : Instance-level graph for Dark Academia. Instance nodes (purple diamonds) connect to category nodes (ochre dots), showing concrete examples of elements, emotions, and behaviours associated with the aesthetic.

To make the creative phases clearer at the instance level, we separated instance nodes by process phase (Divergence, Convergence, Metaconvergence) and connected each instance to the dataset activities that instantiate those phases. The diagram below demonstrates how Dark Academia maps to specific phase activities e.g., Inspiration, Bricolage, Ideation in Divergence; Synthesis, Selection in Convergence; and Recontextualization, Reproduction in Metaconvergence.

Figure 5 : Process-phase mapping for Dark Academia. Instances are grouped by Divergence / Convergence / Metaconvergence, showing how concrete activities and elements realise each phase.
